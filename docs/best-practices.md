# Хорошие практики и приемы

Ниже — практики, которые помогают получать стабильный результат и экономить токены.

---

## 1. Формат запроса: цель → ограничения → критерии
- Цель: что именно нужно сделать.
- Ограничения: какие файлы/модули трогать нельзя.
- Критерии: как понять, что задача решена.

## 2. Минимально достаточный контекст
- Передавать только нужные файлы/фрагменты.
- Удалять лишнее из контекста.
- Помнить про цену входных токенов и кэш.

## 3. Маленькие итерации
- Делить задачу на шаги.
- После каждого шага проверять результат.

## 4. Явные ожидания к формату ответа
- "Дай план", "объясни", "сделай патч".
- Отдельно просить про тесты и риски.

## 5. Контроль области изменений
- Указывать точные файлы.
- Просить минимальные правки.
- Балансировать размер и количество файлов (слишком много/большие — дорого).

## 6. Режим ревью
- Просить объяснить изменения.
- Искать потенциальные баги и регрессии.

## 7. Экономия токенов
- Короткие запросы.
- Четкие формулировки.
- Не повторять весь контекст.
- Чаще начинать новый чат, чем тащить длинную историю.

## 8. Шаблоны и переиспользование
- Хранить лучшие промпты.
- Делать "боевой" набор для типовых задач.

## 9. Проверка качества
- Тесты, линтеры, ручная проверка.
- Быстрый просмотр диффа.

## 10. Подстройка под роль
- Для бэкэнда: акцент на API, модели, тесты.
- Для фронта: UI, состояние, стили.
- Для дизайна: структура, ограничения, контент.

## 11. Выбор модели под задачу
- Быстрые модели — для простых задач и черновиков.
- Сложные задачи — только "умные" модели.
