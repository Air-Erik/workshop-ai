# Шаблоны промптов

Ниже — шаблоны для типовых задач. Заполняйте поля в фигурных скобках.

---

## 1. Рефакторинг
**Цель:** улучшить качество кода без изменения поведения.

**Шаблон:**
```
Цель: рефакторинг {модуль/файл}, сохранив поведение.
Контекст: {файлы/фрагменты}.
Ограничения: {что нельзя менять}.
Критерии: {как проверить}.
Формат ответа: план, затем патч.
```

---

## 2. Багфикс
**Цель:** найти и исправить дефект.

**Шаблон:**
```
Цель: исправить баг {описание}.
Контекст: {стек/файлы/логи/шаги воспроизведения}.
Ограничения: {что нельзя менять}.
Критерии: {какие тесты/проверки должны пройти}.
Формат ответа: объяснение причины + патч.
```

---

## 3. Генерация тестов
**Цель:** покрыть код тестами.

**Шаблон:**
```
Цель: добавить тесты для {функция/класс}.
Контекст: {код и требования}.
Ограничения: {фреймворк тестов, стиль}.
Критерии: {что считается покрытым}.
Формат ответа: список тест-кейсов + патч.
```

---

## 4. Документация
**Цель:** описать модуль/эндпоинт.

**Шаблон:**
```
Цель: написать документацию для {модуль/эндпоинт}.
Контекст: {код, интерфейсы, примеры}.
Ограничения: {стиль, формат}.
Критерии: {что должно быть отражено}.
Формат ответа: markdown.
```

---

## 5. Дизайн/контент (для дизайнеров и маркетинга)
**Цель:** создать текст/структуру.

**Шаблон:**
```
Цель: подготовить {контент/структуру}.
Контекст: {аудитория, тон, ограничения бренда}.
Ограничения: {формат, объем}.
Критерии: {что считать хорошим результатом}.
Формат ответа: {список/таблица/markdown}.
```
