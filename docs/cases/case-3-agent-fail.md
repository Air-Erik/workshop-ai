# Кейс 3: Агент провалил задачу из-за плохой постановки

## Контекст задачи
- Нужно внести правки в несколько файлов.
- Задача сформулирована слишком общо.

## Плохой промпт
- Смешаны цели, нет критериев.

## Исправленный промпт
- Декомпозиция + четкие критерии + ограничения по файлам.

## Результат
- Агент делает минимальные, проверяемые правки.

## Выводы
- Лучше 3 маленьких задачи, чем одна большая.
