# Типичные ошибки и паттерны решения

Ниже — частые ошибки при работе с ИИ-агентами и практичные способы их исправлять.

---

## 1. Непонимание принципов работы моделей
**Симптомы**
- "Модель тупит", "ответы случайные".
- Повторы одного и того же запроса.

**Почему так**
- Не учтены контекст и ограничения окна.
- Нет явной цели и критериев качества.

**Паттерн решения**
- Сформулировать цель, ограничения, критерии приемки.
- Дать минимально достаточный контекст.

---

## 2. Непонимание ценообразования и лимитов
**Симптомы**
- Лимит "сгорает" за 1–2 недели.
- Длинные диалоги без результата.

**Почему так**
- Много лишнего контекста.
- Нет стратегии экономии токенов.

**Паттерн решения**
- Использовать короткие итерации.
- Ограничивать область изменений.
- Переиспользовать шаблоны промптов.

---

## 3. Незнание инструментов Cursor
**Симптомы**
- Все задачи решаются через Chat.
- Игнорирование Agent/Composer.

**Почему так**
- Неочевидна разница режимов.
- Нет понимания, где какой режим эффективнее.

**Паттерн решения**
- Быстрое сопоставление: Chat = обсуждение, Composer = правки, Agent = задача.
- Явно указывать файлы и контекст.

---

## 4. Плохая постановка задачи
**Симптомы**
- Ответы "не то".
- Много времени на правки.

**Почему так**
- Запрос слишком общий.
- Нет критериев и примеров.

**Паттерн решения**
- Описывать входные данные, ожидаемый результат и ограничения.
- Добавлять примеры "как должно быть".

---

## 5. Смешивание нескольких задач в одной
**Симптомы**
- Результаты поверхностные.
- Ошибки и недопонимания.

**Почему так**
- Модель теряет фокус.

**Паттерн решения**
- Декомпозировать задачи.
- Проверять по шагам.

---

## 6. Принятие ответа без проверки
**Симптомы**
- Баги в проде.
- Сюрпризы на ревью.

**Почему так**
- Слепое доверие.

**Паттерн решения**
- Минимальный набор проверок: чтение, тесты, линтер.
- "Режим ревью": просить объяснить правки.

---

## 7. Отсутствие "stop conditions"
**Симптомы**
- Бесконечные попытки.
- Ухудшение результата.

**Почему так**
- Нет критериев остановки.

**Паттерн решения**
- Ограничить число итераций.
- Фиксировать критерии успеха заранее.
